## 1. Что это и как работает

Notify Agent для Windows — тот же агент уведомлений что работает на Linux, портированный на платформу Windows. Логика приёма и отображения уведомлений полностью идентична — отличается только то, как агент запускается, хранится и стартует автоматически.

Агент:
- Поднимает HTTP-сервер на порту **9988**
- Принимает JSON-запросы с заголовком, текстом, уровнем тревоги и типом окна
- Показывает окно на экране — полноэкранное (блокирует работу) или попап (не мешает)
- Сидит в системном трее всё время работы компьютера

## 2. Архитектура

```
Браузер администратора / Ansible-плейбук / curl
                  │
                  ▼
        Сервер панели :8080
        (работает на Linux, без изменений)
                  │
                  │  HTTP POST :9988
                  ▼
        NotifyAgent.exe  (Windows)
                  │
                  ▼
        Окно на экране пользователя
        Fullscreen  или  Popup
```

Один и тот же сервер панели управления и один и тот же Ansible-плейбук могут одновременно слать уведомления и на Linux, и на Windows машины — формат запроса не отличается.

## 3. Нужен ли Python на рабочих станциях

**Нет.** Python требуется только один раз — на машине, где собирается `.exe` файл. На целевых рабочих станциях Python устанавливать не нужно.

### Почему

`PyInstaller` берёт код агента и весь интерпретатор Python вместе со всеми зависимостями (PyQt5 и стандартная библиотека) и упаковывает в **один самодостаточный файл** — `NotifyAgent.exe`. Внутри уже всё необходимое для запуска.

### Таблица по этапам

| Этап | Где происходит | Python нужен? |
|---|---|---|
| Сборка `.exe` | Машина администратора | ✅ Да |
| Установка агента | Каждая рабочая станция | ❌ Нет |
| Работа агента | Каждая рабочая станция | ❌ Нет |

## 4. Сборка .exe

Собирается один раз на любой Windows-машине с установленным Python.

### Файлы для сборки

```
agent_windows.py     ← код агента
build.bat            ← скрипт сборки
notify.ico           ← иконка (опционально)
```

### Шаги

```cmd
build.bat
```

Скрипт сам:
1. Проверяет что Python установлен и добавлен в PATH
2. Устанавливает зависимости: `PyQt5`, `pyinstaller`
3. Собирает `dist\NotifyAgent.exe`

Если `notify.ico` отсутствует — соберёт без кастомной иконки, не упадёт с ошибкой.

### Если Python не в PATH

Установщик с python.org → на первом экране отметить галочку **"Add python.exe to PATH"** → Install Now.

Либо вручную через **Параметры системы → Переменные среды → Path**, добавить:
```
C:\Users\ИмяПользователя\AppData\Local\Programs\Python\Python3XX\
C:\Users\ИмяПользователя\AppData\Local\Programs\Python\Python3XX\Scripts\
```

После изменения — обязательно перезапустить командную строку.

## 5. Установка на рабочую станцию

Копируются только два файла:

```
NotifyAgent.exe
install-agent.bat
```

Оба файла — в одну папку (флешка, сетевая папка, любой способ доставки). На целевой машине запустить `install-agent.bat` от имени администратора.

---

## 6. Что именно делает install-agent.bat

| Шаг | Действие | Зачем |
|---|---|---|
| 1 | Создаёт папку `%LOCALAPPDATA%\NotifyAgent\` | Постоянное системное место хранения |
| 2 | Копирует `NotifyAgent.exe` туда | Файл не зависит от исходной папки на рабочем столе |
| 3 | Прописывает `HKCU\Software\Microsoft\Windows\CurrentVersion\Run` | Автозапуск при каждом входе пользователя в систему |
| 4 | Открывает порт 9988 в брандмауэре Windows | Без этого правила Firewall блокирует входящие подключения |
| 5 | Запускает агента сразу | Не нужно ждать перезагрузки чтобы проверить работу |

Без этого скрипта пришлось бы вручную копировать файл, лезть в реестр и настраивать брандмауэр на каждой машине отдельно.

---

## 7. Где хранятся файлы

```
%LOCALAPPDATA%\NotifyAgent\
├── NotifyAgent.exe                          ← исполняемый файл (копия)
├── notify-agent.lock                        ← файл блокировки одного экземпляра
└── notify-agent-<имя_пользователя>.log      ← лог работы агента
```

Реальный путь обычно:
```
C:\Users\Иванов\AppData\Local\NotifyAgent\
```

---

## 8. Можно ли удалять исходные файлы с рабочего стола

**Да.** После того как `install-agent.bat` отработал — `NotifyAgent.exe` и `install-agent.bat` на рабочем столе (или где они лежали изначально) больше не нужны.

Автозапуск в реестре прописан на путь `%LOCALAPPDATA%\NotifyAgent\NotifyAgent.exe` — то есть на скопированную версию, а не на исходный файл. Удаление исходников не повлияет на работу агента.

Проверить что копия на месте перед удалением исходников:
```cmd
dir %LOCALAPPDATA%\NotifyAgent
```

## 9. Проверка работы агента

### Системный трей

Иконка появляется рядом с часами (может быть в скрытых значках — стрелка ^). Правый клик открывает меню с тестовыми кнопками: тест полноэкранного окна, тест попапа, статус, выход.

### Проверка порта

```cmd
netstat -an | findstr 9988
```
Строка с `LISTENING` означает что агент слушает порт и готов принимать уведомления.

### Тестовый запрос

```cmd
curl -X POST http://127.0.0.1:9988 ^
  -H "X-Token: supersecrettoken123" ^
  -H "Content-Type: application/json" ^
  -d "{\"title\":\"Тест\",\"message\":\"Работает!\",\"level\":\"info\"}"
```

При успехе на экране появится попап с текстом «Работает!».

## 10. Различия с Linux-версией

| Параметр | Linux | Windows |
|---|---|---|
| Среда выполнения | Системный Python + PyQt5 | Зашито внутри `.exe` |
| Путь установки | `/opt/notify-agent/` | `%LOCALAPPDATA%\NotifyAgent\` |
| Автозапуск | systemd user service + XDG autostart | Реестр `HKCU\...\Run` |
| Блокировка одного экземпляра | `fcntl.flock` | `msvcrt.locking` |
| Брандмауэр | Обычно не требуется | Правило в Windows Firewall |
| Позиционирование попапа | Несколько повторных попыток (X11/Wayland) | Одна попытка — Windows точно ставит окно с первого раза |
| Формат JSON-запроса | Идентичен | Идентичен |
| Логика типов окон | Идентична | Идентична |

## 11. Массовое развёртывание

### Active Directory GPO
Скрипт входа (logon script) в групповой политике запускает `install-agent.bat` при входе пользователя в домен. Подходит для крупных инфраструктур с централизованным управлением.

### PowerShell Remoting
```powershell
$computers = Get-Content computers.txt
Invoke-Command -ComputerName $computers -FilePath install-agent.ps1
```
Требует предварительно настроенного WinRM на целевых машинах.

### Сетевая папка
Положить `NotifyAgent.exe` и `install-agent.bat` в общую сетевую папку, сотрудники запускают установщик самостоятельно. Подходит для небольших организаций без сложной автоматизации.

## 12. Удаление агента

```cmd
uninstall-agent.bat
```

Выполняет:
- Останавливает процесс `NotifyAgent.exe` (`taskkill`)
- Удаляет запись автозапуска из реестра
- Удаляет правило брандмауэра
- Удаляет папку `%LOCALAPPDATA%\NotifyAgent\` целиком

---

## 13. Пересборка после изменений

Если нужно изменить логику агента (текст кнопок, цвета, токен авторизации):

1. Отредактировать `agent_windows.py`
2. Запустить `build.bat` на машине с Python заново
3. Получить новый `NotifyAgent.exe` в папке `dist\`
4. Разослать обновлённый `.exe` — `install-agent.bat` перезапишет старую версию при повторном запуске

## 14. Диагностика проблем

### Каракули в консоли вместо текста
Кодировка консоли CP866 не совпадает с UTF-8 скрипта. Решение — добавить `chcp 65001` в начало `.bat` файла или использовать только латиницу в самих `.bat` скриптах.

### Build.bat завершается без ошибок, но exe не появился
Проверить `dist\` папку — PyInstaller создаёт `build\` (временные файлы) и `dist\` (готовый результат) отдельно. Нужный файл всегда в `dist\`.

### curl: Failed to connect to port 9988
Агент ещё не успел запуститься (подождать 3-5 секунд после установки) либо порт блокируется антивирусом — проверить исключения в антивирусном ПО отдельно от брандмауэра Windows.

### Иконка не появляется в трее
Windows может скрывать новые иконки трея по умолчанию — проверить стрелку "показать скрытые значки" рядом с часами, либо настроить отображение через **Параметры → Персонализация → Панель задач → Выбрать значки для отображения**.

### Агент не стартует после перезагрузки
Проверить что запись в реестре существует:
```cmd
reg query "HKCU\Software\Microsoft\Windows\CurrentVersion\Run" /v "NotifyAgent"
```
Если запись отсутствует — переустановить через `install-agent.bat`.
